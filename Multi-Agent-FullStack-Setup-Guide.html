How it works:

  • The Lead Agent runs in your interactive Claude Code session
  • Sub-Agents are spawned as headless processes via claude -p
  • Each agent reads its role from .claude/agents/[role].md and uses skills from .claude/skills/
  • Tasks are routed by layer: [DB] → Database Engineer, [API] → DotNet Architect, [UI] → Angular Expert
  • The Lead Agent validates build + test results before proceeding to the next phase
  • ⚠️ Git and deployment operations always require your explicit approval

6. Development Lifecycle

🔄 End-to-End Lifecycle — Follow this order:
  1. Plan (/agent-plan) → Interview + spec + approval loop
  2. UI/UX Design (/agent-design) → Screen specs with Tailwind classes — if feature has UI
  3. Design Docs (/agent-design-docs) → HTML files: Architecture, API Spec, Technical Design — BEFORE coding
  4. Chunk Loop (repeat per chunk):
    • /agent-build → Routes to DB Engineer / DotNet Architect / Angular Expert per task layer
    • /agent-review → Cross-stack review (security, bugs, quality, perf, tests)
    • /agent-test → Unit test verification + coverage check
  5. API Tests (/agent-api-tests) → Feature-wide integration tests — AFTER all chunks
  6. E2E Tests (/agent-e2e-tests) → Feature-wide Playwright tests — AFTER all chunks
  7. Final Docs (/agent-final-docs) → HTML files: Summary, coverage, API docs, release notes, deployment instructions, retro
  8. Git Commit (/agent-commit) → ⚠️ Asks you before stage, commit, and push
  9. Deploy (/agent-deploy) → ⚠️ Asks you before preview AND production deployment to Vercel

8. Step-by-Step Workflow

Phase 1: Plan the Feature

/agent-plan user-management

The Lead Agent asks you 13 questions across 5 rounds (Context, Requirements, UI, Constraints, Confirmation). Your answers are passed to the Planner Agent which generates the full spec and task breakdown.

Approval loop: Review the plan, give feedback ("split T02", "add caching"), or say "approved" to lock it.

Phase 2: UI/UX Design (if feature has UI)

/agent-design user-management

The UI/UX Designer creates component specs with exact Tailwind CSS classes, all states (loading, empty, error, success), responsive breakpoints, and accessibility requirements. These specs guide the Angular Expert during implementation.

Phase 3: Design Documents (HTML)

/agent-design-docs user-management

Generates three self-contained HTML files in docs/user-management/: Architecture Design, API Specification, and Technical Design. Open directly in any browser. Stakeholders review these before coding starts.

Phase 4: Build → Review → Test (per chunk)

# Build chunk 1 (DB + API tasks)
/agent-build user-management T01 to T03

# Review chunk 1 files
/agent-review user-management chunk-1

# Verify chunk 1 unit tests
/agent-test user-management chunk-1

# Build chunk 2 (UI tasks)
/agent-build user-management T04 to T06

# Review + Test chunk 2
/agent-review user-management chunk-2
/agent-test user-management chunk-2

The build command automatically routes [DB] tasks to Database Engineer, [API] tasks to DotNet Architect, and [UI] tasks to Angular Expert.

Phase 5: Feature-Wide API & E2E Tests

/agent-api-tests user-management
/agent-e2e-tests user-management

Runs only after ALL chunks are complete. Tests cross-component integration and end-to-end user flows.

Phase 6: Final Documentation (HTML)

/agent-final-docs user-management

Generates six self-contained HTML files in docs/user-management/: implementation summary, test coverage, API docs, release notes, deployment instructions (DB migrations + backend + Vercel frontend + rollback plan), and retrospective — all based on what was ACTUALLY built.

Phase 7: Git Commit & Push (⚠️ asks you first)

/agent-commit user-management

The Lead Agent shows you the exact files to stage, the commit messages (conventional commit format), and the branch name. You must explicitly approve each step — branch creation, staging, each commit, and push. Nothing happens without your "yes".

Phase 8: Deploy to Vercel (⚠️ asks you first)

/agent-deploy user-management

The Lead Agent shows you the pre-deployment checklist and generates vercel.json. You must explicitly approve both preview deployment and production deployment. If anything fails, the agent stops and shows you the error.

Phase 9: Final Status

/agent-status user-management

Shows complete feature dashboard: all 9 phases, chunk status, task status, documentation, git commits, and deployment state.

🎉 Done! Full-stack feature complete — designed, built, tested, documented, committed, and deployed.

12. Best Practices

✅ Getting Started

  • Use phase-by-phase commands first before trying /orchestrate
  • Always review the Planner's output before approving
  • Check claude-notes/agent-log.md after each run

✅ Task Organization

  • Order tasks by dependency: Database → Backend API → Frontend UI
  • Keep chunks to 3-5 tasks max for manageable review cycles
  • For large features (10+ tasks), split into sub-features

✅ Context Window Management

  • Sub-agents run in separate context windows via claude -p
  • Build in batches for large features: T01-T04 then T05-T08
  • Use /agent-status to resume after breaks

✅ Customizing for Your Team

  • Edit agent .md files to add your team's conventions and rules
  • Update CLAUDE.md with your specific build/test/lint commands
  • Add project-specific patterns to the skills library
